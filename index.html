Multiple Stack Architecture

Intelligent vehicle system is complicated and a good architecture to start with will help us manage the complexity. We propose a multiple stack layered architecture that divides the functions according to different ways of interaction with the world and also has top-down feature with well-defined interfaces in between. Model Checking

Before we allow these systems on public highways we must prove that collaborative driving systems will do no harm even when rare, multiple failures occur. To verify that such systems are safe, we apply the techniques developed for communications protocols. View details »

Conformance Testing

Conformance testing is the testing procedure that checks whether an implementation correctly conforms to the specification. We apply this on testing of intelligent vehicles after the model checking, and together can alleviate the complexity of pair-wise testing when collaboration between systems exists. The timing issue is the most challenging part and we can come up with a good solution when we introduce using absolute time instead of internal timers. View details »
