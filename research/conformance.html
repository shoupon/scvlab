Multiple Stack Architecture

Multiple Stack Architecture Synchronous Protocol Design Driver-assisted Cooperative Merge Protocol Model Checking Conformance Testing

Conformance testing is the testing to determine whether a product or system implemented by manufacturer conforms to the specification. This technique has been used in protocol testing for communication systems in order to ensure that equipment from various manufacturers conform to a standard and thus can interoperate correctly.

Our proposed multiple stack architecture for intelligent vehicles partition the complex system into smaller pieces, which are eligible for using conformance testing techniques via the well-defined interfaces. The control portion of the protocol is usually specified as a finite state machine, and a test sequences are generated to check the conformity to the specification of the protocol.

Time-critical events are very common in collaborative intelligent driving protocols. The postman algorithm that was used for communications protocols does not consider time critical events. In communication systems, timers are used to keep track of time-triggered events. Using internal timers not only impose constraints on generating test sequences, but also increase the test time for waiting the timer to expire. We use the strategy of using accurate clocks supported by synchronized clocks in the timing stack. A shared memory between processes stores the time of occurrence for all time related events, hence for the module that has time-related events, timeouts are set as outputs/inputs to/from the shared memory. To test an edge, the input to the module is just the input that is received from the shared memory.

Our research interests include studying the potential improvements by using external timers instead of internal timers. We will compare the lengths of UIO sequences, and determine the relationship of two strategies. We would also like to study the scalability of the improvement if it exists.
