Multiple Stack Architecture

Multiple Stack Architecture Synchronous Protocol Design Driver-assisted Cooperative Merge Protocol Model Checking Conformance Testing

We proposed a multiple stack architecture for intelligent vehicles. Intelligent vehicles are similar to communication networks in the aspect that both systems interact with the physical world using rapidly evolving devices. However, intelligent vehicles are much more complicated since they interact with the physical world in several ways, and some of them are time-critical events. Our proposed architecture not only has layered feature as in communication architecture, but also adopts multiple stacks to solve this problem.

We organize different ways of interacting with the physical world into separate stacks, and within each stack there are several layers with well-defined interfaces. In each stack, modules in lower layers are closer to the physical world and provide services to the layer that resides one layer higher. Besides that, modules can also provide services to ones in other stacks.

By having well defined interfaces between the layers in the stacks we can change one particular layer without affecting the entire stack as in communication architecture. The layered architecture with services interfaces allows us to apply model checking and conformance testing to individual components. The interfaces between layers also eliminate physical constraints when generating testing sequence for each component. The test sequences can be injected into the components through the well-defined interface, and the outputs can be easily observed from the interface.
